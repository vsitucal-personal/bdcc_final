Executive Summary

This report explores the prediction of win or loss outcomes in Clash Royale dataset with 130M matches filtered down to 1v1 ranked matches with 12M games. Each game record contains information on player trophies and the decks of cards used. Feature engineering was done focused on deriving a "net score" for each card, representing its average contribution to crowns earned, along with other features such as trophies and card descriptions to enhance predictive modeling with SparkML. The dataset is well-balanced, with win and loss outcomes distributed evenly, with PCC (Proportional Chance Criterion) value of 50%, benchmark accuracy to beat is calculated as 1.25 times the PCC at 62.5% accuracy.

SparkML models used were LogisticRegression, RandomForestClassifier, GBTClassifier and LinearSVC. Initial best model was Linear Support Vector Classifier (LinearSVC) which achieved 99% accuracy, but this performance was primarily driven by player trophies, identified as a dominant predictor. To eliminate this dependency, models were retrained after removing player trophies as a feature. Without player trophies, the Gradient Boosted Trees Classifier (GBTClassifier) emerged as the best-performing model, achieving a robust 94% accuracy. The top predictor in this revised model was features related to the sum of the net scores of the cards in each player's deck, underscoring the importance of card composition in determining match outcomes.

Introduction

Clash Royale a game developed by Supercell, is a real-time strategy game that combines card collecting, tower defense, and multiplayer battles with 13 million average monthly active players in 2022. Players deploy cards to attack opponent's towers and defend their own. a game is won by destroying the opponents towers while protecting your own. Each player has three towers: one King Tower and two Crown Towers. Destroying the opponents King Tower results in an automatic victory, regardless of the status of the Crown Towers. If the King Tower remains standing, the match is decided by the number of Crown Towers destroyed within the three-minute regulation time. In the event of a tie, the game enters a sudden-death overtime period, where the first player to destroy any tower wins. If no towers are destroyed during overtime, the player with the most damage dealt to towers is declared the winner. This structure encourages a blend of aggressive strategies to seize victory and defensive tactics to mitigate losses, adding depth to the competitive gameplay. Matches demand quick thinking and tactical precision

Problem Statement

The central question of this report is: What features best determine triumph or defeat in Clash Royale 1v1 ranked matches? By analyzing player attributes, deck composition, and engineered features, the goal is to identify the most influential factors driving match outcomes.

Data Prep and Cleaning

Filters:

  • Ranked 1v1 Game Mode Only
  • Filter out duplicate rows with dropDuplicates (removes rows with all identical column values)
  • Filter out ties
  • Games filtered to percent_diff_trophies < 0.50
In[]:
df = (df.filter(df.gamemode == "72000323")
    .dropDuplicates()
    .withColumn(
        "p1_win",
        F.when(F.col("p1_crowns") > F.col("p2_crowns"), 1)
         .when(F.col("p1_crowns") < F.col("p2_crowns"), -1)
         .otherwise(0)
    )
    .filter(F.col("p1_win") != 0)
    .withColumn("datetime", F.to_timestamp("datetime", "yyyyMMdd'T'HHmmss.SSS'Z'"))
    .withColumn("abs_diff_trophies", F.abs(F.col("p1_trophies")-F.col("p2_trophies")))
    .withColumn(
        "percent_diff_trophies",
        (F.abs(F.col("p1_trophies") - F.col("p2_trophies")) / ((F.col("p1_trophies") + F.col("p2_trophies")) / 2))
    ).filter(F.col("percent_diff_trophies") < 0.50)
    .withColumn(
        "winner_trophies",
        F.when(F.col("p1_win") == 1, F.col("p1_trophies"))
         .otherwise(F.col("p2_trophies"))
    ).withColumn(
        "loser_trophies",
        F.when(F.col("p1_win") == -1, F.col("p2_trophies"))
         .otherwise(F.col("p1_trophies"))
    )
)

Feature selection

Features and Class labels are described below

  • p1_win - if p1 has more crowns than p1
  • abs_diff_trophies - absolute difference of p1_trophies and p2_trophies
  • percent_diff_trophies - absolute percentage difference of p1_trophies and p2_trophies
  • winner_trophies - p1_trophies if p1 is winner else p2_trophies
  • loser_trophies - p1_trophies if p1 is loser else p2_trophies

In the query below is used to calculate a "net score" associated to a specific card. It is determined by looking at a history of games and working out on average how much a card has contributed in gaining a crown. Calculated netscores are based on matches dated 2023-07 and earlier.

ranked_cardscores = spark.sql('''
SELECT up.card, c.name, c.elixir, c.type, c.rarity, c.is_evolved,
    COUNT(1) lineups_played,
    AVG(CASE WHEN up.player = 'p1' THEN p1_crowns
        ELSE p2_crowns END) crowns_generated,
    AVG(CASE WHEN up.player = 'p1' THEN p2_crowns
        ELSE p1_crowns END) crowns_allowed,
    AVG(CASE WHEN up.player = 'p1' THEN p1_crowns - p2_crowns
        ELSE p2_crowns - p1_crowns END) net_score
FROM data UNPIVOT (
    card for player in (
        p1_card1 as p1, p1_card2 as p1, p1_card3 as p1, p1_card4 as p1,
        p1_card5 as p1, p1_card6 as p1, p1_card7 as p1, p1_card8 as p1,
        p2_card1 as p2, p2_card2 as p2, p2_card3 as p2, p2_card4 as p2,
        p2_card5 as p2, p2_card6 as p2, p2_card7 as p2, p2_card8 as p2
    )
) up
INNER JOIN cards c ON up.card = c.id
WHERE gamemode = 72000323 -- Ranked1v1 only
    AND CAST(datetime AS DATE) <= '2023-07-31'
GROUP BY up.card, c.name, c.elixir, c.type, c.rarity, c.is_evolved
''')

def compute_pcc(df):
    df.groupBy("p1_win").count().createOrReplaceTempView('pcc_base')
    return spark.sql("""
    WITH total_count_cte AS (
        SELECT SUM(count) AS total_count FROM pcc_base
    )
    SELECT 
        SUM(POWER(r.count / t.total_count, 2)) AS pcc
    FROM 
        pcc_base r, total_count_cte t
    """).collect()[0][0]

def run_models(ml_model, prep_df, return_results=False, show_importance=False):
    print(f"Running {ml_model.__class__.__name__}")
    feature_list = [i for i in prep_df.columns if all(substring not in i for substring in ["p1_win", "datetime"])]
    
    assembler = VectorAssembler(
        inputCols=feature_list,
        outputCol="features"
    )

    df_va = assembler.transform(prep_df)
    
    win_indexer = StringIndexer(inputCol="p1_win", outputCol="label")

    win_indexer_model = win_indexer.fit(df_va)
    df_va = win_indexer_model.transform(df_va)
    
    df_training = df_va.filter(F.col('datetime') <= '2023-07-31')
    df_test = df_va.filter(F.col('datetime') > '2023-07-31')
    
    scaler = StandardScaler(inputCol="features", outputCol="scaledFeatures")

    labelConverter = IndexToString(
        inputCol="prediction",
        outputCol="predictedLabel",
        labels=win_indexer_model.labels
    )

    pipeline = Pipeline(
        stages=[
            scaler,
            ml_model,
            labelConverter
        ]
    )

    model = pipeline.fit(df_training)
    train_predictions = model.transform(df_training)
    train_predictions = train_predictions.withColumn(
        "predictedLabel", 
        train_predictions["predictedLabel"].cast(IntegerType())
    )

    test_predictions = model.transform(df_test)
    test_predictions = test_predictions.withColumn(
        "predictedLabel", 
        test_predictions["predictedLabel"].cast(IntegerType())
    )

    ml_model = model.stages[1]
    if ml_model.__class__.__name__ in ["LinearSVCModel", "LogisticRegressionModel"]:
        importances = ml_model.coefficients.toArray()
    else:
        importances = ml_model.featureImportances.toArray() 
    
    feature_names = feature_list
    importance_data = [(feature, float(importance)) for feature, importance in zip(feature_names, importances)]
    importance_df = spark.createDataFrame(importance_data, schema=["Feature", "Importance"])
    
    if show_importance:
        importance_df.orderBy(F.abs(F.col('Importance')).desc()).show(100, truncate=False)
    
    train_evaluator = MulticlassClassificationEvaluator(
        labelCol="label", predictionCol="prediction", metricName="accuracy")
    train_accuracy = train_evaluator.evaluate(train_predictions)
    print(f"Train acc: {train_accuracy}")

    test_evaluator = MulticlassClassificationEvaluator(
        labelCol="label", predictionCol="prediction", metricName="accuracy")
    test_accuracy = test_evaluator.evaluate(test_predictions)
    print(f"Test acc: {test_accuracy}")
    
    if not return_results:
        return None
    else:
        return ml_model, train_predictions, test_predictions, importance_df

The decision boundary observed in the plot effectively delineates the two classes, suggesting a clear relationship between the features p1_trophies and p2_trophies in predicting match outcomes. This finding underscores the importance of these features in the model's decision-making process.

The strong performance of the LinearSVC model aligns with this observation, as linear classifiers tend to perform well when features are linearly separable or exhibit a straightforward relationship. The model's relatively high accuracy indicates that p1_trophies and p2_trophies likely contribute significantly to the prediction of match outcomes.

Interestingly, the feature importance reveals an intriguing insight: a lower-rated player often prevails over a higher-rated opponent. This pattern might suggest the presence of a "challenger advantage." Lower-rated players may be underestimated by their higher-rated counterparts, leading to under-preparation or overconfidence in matches. Alternatively, the data could reflect strategic matchups or hidden factors such as differing levels of motivation, risk-taking, or psychological pressure.

To bring out other factors that may affect match outcome we may opt to drop this feature and look at the next top predictors.
